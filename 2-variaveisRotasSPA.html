Módulo 1: Fundamentos da SPA

Neste módulo, vamos explorar os três pilares para construir uma aplicação de página única (SPA) estruturada e estilizada. Abordaremos o conceito de "páginas" como componentes, como criar um sistema de design consistente com variáveis de CSS, e o mais importante: como implementar a navegação entre essas páginas sem recarregar o navegador, o que é a essência de uma SPA, utilizando um sistema de rotas.

O Conceito de SPA e 'Páginas' em React

Em uma SPA, "páginas" são na verdade componentes diferentes que são renderizados dinamicamente na tela. A prática comum é organizar esses componentes de página em uma pasta, como src/pages, para manter a estrutura do projeto clara e intuitiva.

Estilização Consistente e Escalável

Para manter a consistência visual, definimos estilos globais e variáveis de CSS em um arquivo base (src/index.css). Isso cria um "design system" básico, permitindo que a paleta de cores, fontes e espaçamentos sejam reutilizados em toda a aplicação, garantindo uma identidade visual coesa e facilitando a manutenção.

Navegação em uma SPA: O Papel das Rotas

A navegação em uma SPA acontece sem recarregar a página, proporcionando uma experiência de usuário rápida e fluida. Isso é gerenciado por uma biblioteca de roteamento (como a react-router-dom). Ela lê a URL do navegador e, com base no caminho (ex: "/sobre"), renderiza o componente de página correspondente de forma condicional, simulando a navegação tradicional.
